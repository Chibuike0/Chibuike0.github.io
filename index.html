Power BI Project

A PowerBI Dasboard depicting the online shopping across various major cities in United States of America. The number of purchase accross the cities, average memberships by different genders and how satisfied they are where shown on this dashboard. With tree diagram we can take a deeper look into the most satisfied customer cities and least satisfied with their experiences.

Many purchasers did not have a wonderful experience despite having discounts applied to their purchase. Deep diving into the data shows that female customers are more satisfied with thier purchase than male counterparts despite the female having occupied the three memberships than men having only just gold and silver. The trend is that male customers are mostly from south and the female customer are predominantly from northern part of USA. Having said that, the younger the customer, the likely for them to be least satisfied, therefore, improving experiences that cuts across all ages will increase the satisfaction levels.

Python ML project-2

Delving into the IMDb dataset, a collection of 50,000 movie reviews, for sentiment analysis. The OS library and Pandas were used to create a DataFrame, a valuable source of movie reviews and associated sentiments. The aim is to build a machine learning model capable of swiftly determining whether a movie review expresses positive or negative sentiment based on critics' reviews, with the goal of analyzing reviews within seconds.

After confirming data types and checking for missing values and data imbalance, we preprocessed the text data, removing special characters, URLs, stopwords, and expanding contractions. For exploratory analysis, visualized positive and negative keywords using WordCloud, plotted the density of average words used in sentiment, and performed lemmatization using NLTK's WordNetLemmatizer. then bagged the 30 most frequent words into vector format and performed TD-IDF using sklearn.

Finally, built and compared the accuracy of four models: Random Forest, Decision Tree, ADA Boost, and TensorFlow. The Random Forest model, an ensemble learning method, showed promise due to its ability to handle diverse data types and reduce overfitting. Click on the picture for GitHub.

Cloudera-Hadoop-SQL project

Developed a data ingestion and analysis pipeline that involves downloading CSV files, uploading the data into MySQL, exporting the data to HDFS (Hadoop Distributed File System), and performing data analysis using Hive. The pipeline will enable efficient data processing and analytics on large datasets using the Hadoop ecosystem.. Click on the picture for GitHub.

Ingested the sale_data.csv into a MySQL database table on Cloudera. To ensure no duplicate table existed, logged into MySQL in Cloudera terminal using the syntax: "MySQL -u root -p". After accessing MySQL, I checked existing databases with "show databases". Following this, created a new database and table with the same schema as sale_data.csv before ingestion.

This project showcases the flexibility of Hadoop frameworks, allowing smooth transitions between them. Despite challenges in analyzing data with Hive, the data was successfully imported into HDFS on Cloudera, and then into Hive for analysis. To ensure compatibility, the MySQL table was transferred to HDFS, and a corresponding table schema was created in Hive. Using Sqoop, a tool for importing/exporting bulk data, sales_csv was transferred to HDFS with the command "sqoop import â€“connect". It's important to execute this command in a Cloudera HDFS terminal, not the MySQL terminal. The import, split into two using the ID as the primary key, was successful, and the data was ingested into HDFS. The data was then moved from HDFS to Hive for analysis, leveraging Sqoop's ability to seamlessly move data between Hadoop frameworks. This project generated various insights through different analytical queries.
